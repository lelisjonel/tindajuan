# TindaJuan Mobile MVP Wireframe

This folder contains the first low-fidelity wireframe for the TindaJuan MVP.

## Files

- `mobile-mvp-wireframe.html` — mobile-first MVP screens
- `tablet-desktop-wireframe.html` — tablet cashier and desktop owner/admin layouts

## Covered Mobile Screens

1. Store Setup
2. Benta / Checkout
3. Payment Sheet
4. Services / GCash & Maya
5. Service Transaction Form
6. Paninda / Products
7. Add/Edit Product
8. Suki / Utang
9. Customer Detail / Ledger
10. Kaha
11. Daily Reports
12. Menu / Settings

## Design Intent

The wireframe is **mobile-first** because target users may use phones or tablets more than desktop/laptop.

Primary bottom navigation:

```txt
Benta
Services
Paninda
Suki
Menu
```

Secondary screens inside `Menu`:

```txt
Kaha
Reports
Settings
Backup/Export later
```

## Core UX Rules Reflected

- Benta and Services are fast-access screens.
- Cart is sticky on the Benta screen.
- Payment sheet supports Cash, GCash, Maya, mixed payment, and Kulang/Utang.
- If checkout has remaining balance, user should select a Suki.
- GCash/Maya product payments affect wallet balances, not physical kaha.
- Service form shows cash/wallet/income effects before saving.
- Kaha focuses on expected cash drawer balance.
- Reports separate product sales from service volume.
- Personal kuha affects kaha, not product profit.
